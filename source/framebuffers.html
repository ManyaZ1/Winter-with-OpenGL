Framebuffers και FrameBuffer Objects (FBO)

Ένας άλλος τύπος object που απαντάται στην OpenGL —εκτός από τα VAO και VBO που είδαμε στο παρελθόν— είναι τα FrameBuffer Objects (FBO). Αυτά μας δίνουν τη δυνατότητα να ζωγραφίσουμε τη σκηνή, χωρίς να δείξουμε το αποτέλεσμα στην οθόνη. Αυτό, εκ πρώτης όψεως, φαίνεται μάλλον άχρηστο. Παρόλα αυτά, αν δούμε τις δυνατότητές του, εύκολα θα γίνει αντιληπτό γιατί η γνώση χρήσης των FBO είναι απαραίτητη για την υλοποίηση πολλών περίπλοκων εφέ.

Framebuffer

Στο παρελθόν, εν αγνοία μας, χρησιμοποιούσαμε ήδη έναν framebuffer: τον προεπιλεγμένο (default) framebuffer. Ένας framebuffer, όπως υπονοεί και το όνομά του, είναι μία περιοχή στη μνήμη της GPU, η οποία περιέχει πληροφορίες που σχετίζονται με τη σχεδίαση ενός καρέ. Συγκεκριμένα, μπορεί να συνδεθεί με τρεις άλλους τύπους buffers:

  • έναν color buffer (ή και περισσότερους)
  • έναν depth buffer
  • έναν stencil buffer

Η GLFW, κατά τη δημιουργία του παραθύρου, δημιουργεί τον default framebuffer και συνδέει σε αυτόν δύο color buffers, έναν depth buffer και έναν stencil buffer. Έχουμε ήδη αναφερθεί στους δύο color buffers του default framebuffer. Ο ένας από αυτούς είναι αυτός ο οποίος φαίνεται στην οθόνη ενώ ο άλλος είναι αυτός επάνω στον οποίο ζωγραφίζουμε. Έπειτα, καλούμε τη συνάρτηση: glfwSwapBuffers και οι color buffers αλλάζουν θέση: αυτός επάνω στον οποίο ζωγραφίζαμε εμφανίζεται στην οθόνη, ενώ αυτός που μέχρι τότε εμφανιζόταν στην οθόνη, είναι αυτός επάνω στον οποίο θα ζωγραφίσουμε το επόμενο καρέ. Η διαδικασία συνεχίζεται για κάθε καρέ.

Επιπλέον, έχουμε μιλήσει για τον depth buffer ή z-buffer. Τον χρησιμοποιήσαμε προκειμένου να υλοποιήσουμε το depth test: μία διαδικασία με την οποία απορρίπτουμε κατά τη σχεδίαση του καρέ fragments τα οποία βρίσκονται πίσω από άλλα που ήδη έχουμε ζωγραφίσει.

Δεν έχουμε αναφερθεί ακόμη ούτε έχουμε χρησιμοποιήσει τον stencil buffer. Για την ώρα, θα πούμε απλά ότι είναι ένας buffer που, όπως ο depth buffer, μας επιτρέπει να επεμβαίνουμε επάνω στα fragments που ζωγραφίζονται με έναν λιγότερο αυτόματο τρόπο, προσφέροντάς μας, όμως, περισσότερες δυνατότητες.

Να αναφέρουμε ακόμη, ότι ένας framebuffer δε θεωρείται πλήρης αν δεν συνδέεται με τουλάχιστον έναν color buffer (με άλλα λόγια, επιπλέον color buffers, ο depth buffer και ο stencil buffer είναι προαιρετικοί).

FrameBuffer Objects (FBO)

Η OpenGL μάς προσφέρει τη δυνατότητα να δημιουργήσουμε τους δικούς μας framebuffers και να συνδέσουμε σε αυτούς όποιους color, depth και stencil buffers επιθυμούμε. Δημιουργούμε ένα FBO ως εξής:

GLuint FBO;
glGenFramebuffers(1, &FBO);

Όπως και τα υπόλοιπα objects, για να ενεργοποιηθεί πρέπει να το κάνουμε bind.

glBindFramebuffer(GL_FRAMEBUFFER, FBO);

Στη συνέχεια, θα δημιουργήσουμε ένα RenderBuffer Object (RBO), που είναι ο τρόπος που η OpenGL μάς επιτρέπει να ορίσουμε δικούς μας color, depth και stencil buffers.

GLuint colorRBO;
glGenRenderbuffers(1, &colorRBO);

Και θα το κάνουμε bind.

glBindRenderbuffer(GL_RENDERBUFFER, colorRBO);

Έπειτα, θα ορίσουμε τις διαστάσεις του RBO.

glRenderbufferStorage(GL_RENDERBUFFER, GL_RGBA8, 800, 600);

Η παραπάνω συνάρτηση λέει πως το RBO θα έχει 4 συνιστώσες (RGBA) από 8 bit η καθεμία και το μέγεθος σε pixel θα είναι 800x600.

Μπορούμε τώρα να κάνουμε unbind το colorRBO, κάνοντας bind το default RBO.

glBindRenderbuffer(GL_RENDERBUFFER, 0);

Το default RBO, FBO, VAO, VBO ή οποιοδήποτε άλλο default object αντιστοιχεί στον κωδικό 0.

Θα επαναλάβουμε την παραπάνω διαδικασία για ένα δεύτερο RBO, το οποίο θα χρησιμοποιήσουμε ως depth buffer.

glGenRenderbuffers(1, &depthRBO);
glBindRenderbuffer(GL_RENDERBUFFER, depthRBO);
glRenderbufferStorage(GL_RENDERBUFFER, GL_DEPTH_COMPONENT16, 800, 600);
glBindRenderbuffer(GL_RENDERBUFFER, 0);

Όπως βλέπουμε, το depthRBO έχει μόνο μία συνιστώσα βάθους μεγέθους 16 bit για κάθε pixel.

Τώρα, με τη συνάρτηση glFramebufferRenderbuffer μπορούμε να συνδέσουμε τα RBO στο FBO που δημιουργήσαμε.

glFramebufferRenderbuffer(GL_FRAMEBUFFER, GL_COLOR_ATTACHMENT0, GL_RENDERBUFFER, colorRBO);
glFramebufferRenderbuffer(GL_FRAMEBUFFER, GL_DEPTH_ATTACHMENT, GL_RENDERBUFFER, depthRBO);

Τέλος, κάνουμε πάλι bind τον default framebuffer.

glBindFramebuffer(GL_FRAMEBUFFER, 0);

Κάθε φορά που ζωγραφίζουμε κάτι, αυτό ζωγραφίζεται στο ενεργό FBO. Έτσι, αν επιθυμούμε να ζωγραφίσουμε ένα μοντέλο στο FBO που ορίσαμε, θα πρέπει πρώτα να χρησιμοποιήσουμε τη συνάρτηση glBindFramebuffer για να ενεργοποιήσουμε το επιθυμητό FBO:

glBindFramebuffer(GL_FRAMEBUFFER, FBO);
// ...
model->draw();
glBindFramebuffer(GL_FRAMEBUFFER, 0);

Εάν εκτελέσουμε τον κώδικα, θα δούμε ότι το μοντέλο model δε φαίνεται στην οθόνη. Αυτό συμβαίνει επειδή ζωγραφίζεται στο FBO και όχι στον default framebuffer.

Ακόμη τα FBO δε φαίνονται ιδιαίτερα χρήσιμα. Αυτό αλλάζει όταν μάθουμε ότι, αντί για RBO, μπορούμε να χρησιμοποιήσουμε χάρτες υφών (textures) ως συνδέσεις ενός FBO. Δηλαδή, μπορούμε, αντί να αποθηκεύουμε τα δεδομένα χρώματος και βάθους (και stencil) που προκύπτουν κατά τη ζωγραφική σε τυχαίους buffers, να τα αποθηκεύουμε σε textures, τα οποία μπορούμε να τα χρησιμοποιήσουμε με οποιονδήποτε τρόπο μπορούν να χρησιμοποιηθούν και οι “κανονικοί” χάρτες υφών.

Σύνδεση texture με FBO

Αντί να δημιουργήσουμε ένα FBO και να το συνδέσουμε με ένα RBO ως color buffer και ένα άλλο RBO ως depth buffer, θα το συνδέσουμε με ένα texture ως color buffer και ένα RBO ως depth buffer.

Αρχικά, δημιουργούμε και ενεργοποιούμε το FBO.

GLuint FBO;
glGenFramebuffers(1, &FBO);
glBindFramebuffer(GL_FRAMEBUFFER, FBO);

Έπειτα, θα δημιουργήσουμε ένα καινούργιο texture και θα το κάνουμε bind.

GLuint colorTexture;
glGenTextures(1, &colorTexture);
glBindTexture(GL_TEXTURE_2D, colorTexture);

Κατ’ αντιστοιχία με τη συνάρτηση glRenderbufferStorage, θα χρησιμοποιήσουμε τη συνάρτηση glTexImage2D για να ορίσουμε ένα texture με 4 συνιστώσες χρώματος (RGBA) από 8 bit η καθεμία και το μέγεθος σε pixel θα είναι 800x600.

glTexImage2D(GL_TEXTURE_2D, 0, GL_RGBA8, 800, 600, 0, GL_RGBA, GL_UNSIGNED_BYTE, NULL);

Προσέξτε ότι το τελευταίο όρισμα, που είναι η εικόνα του texture είναι NULL, επειδή δεν την έχουμε ακόμη διαθέσιμη. Θα δημιουργηθεί κατά τη σχεδίαση του καρέ.

Χρησιμοποιούμε τη συνάρτηση glTexParameteri για να ορίσουμε κάποιες ακόμη παραμέτρους του texture. Με τις εντολές:

glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MIN_FILTER, GL_LINEAR);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_MAG_FILTER, GL_LINEAR);

ενεργοποιούμε (δι)γραμμική παρεμβολή αν χρειαστεί να γίνει μεγέθυνση ή σμίκρυνση του texture, ενώ με τις εντολές:

glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_S, GL_REPEAT);
glTexParameteri(GL_TEXTURE_2D, GL_TEXTURE_WRAP_T, GL_REPEAT);

ορίζουμε να επαναλαμβάνεται το texture εάν βγούμε εκτός των ορίων του κατά τη δειγματοληψία.

Αφού ολοκληρώσουμε τη ρύθμιση του texture, μπορούμε να το απενεργοποιήσουμε.

glBindTexture(GL_TEXTURE_2D, 0);

Θα ακολουθήσουμε την ίδια διαδικασία με πριν, για τον ορισμό ενός RBO βάθους.

glGenRenderbuffers(1, &depthRBO);
glBindRenderbuffer(GL_RENDERBUFFER, depthRBO);
glRenderbufferStorage(GL_RENDERBUFFER, GL_DEPTH_COMPONENT16, 800, 600);
glBindRenderbuffer(GL_RENDERBUFFER, 0);

Τώρα, με τη συνάρτηση glFramebufferTexture2D θα χρησιμοποιήσουμε το colorTextue ως σύνδεση χρώματος για το FBO και το depthRBO ως σύνδεση βάθους.

glFramebufferTexture2D(GL_FRAMEBUFFER, GL_COLOR_ATTACHMENT0, GL_TEXTURE_2D, colorTexture, 0);
glFramebufferRenderbuffer(GL_FRAMEBUFFER, GL_DEPTH_ATTACHMENT, GL_RENDERBUFFER, depthRBO);

Τέλος, θα απενεργοποιήσουμε το FBO και θα ενεργοποιήσουμε πάλι τον default framebuffer.

glBindFramebuffer(GL_FRAMEBUFFER, 0);

Από εδώ και στο εξής, κάθε φορά που ζωγραφίζουμε τη σκηνή έχοντας το FBO που δημιουργήσαμε ενεργοποιημένο, το αποτέλεσμα θα αποθηκεύεται στο colorTexture και θα είναι διαθέσιμο να το χρησιμοποιήσουμε.

Αυτή η δυνατότητα που μας προσφέρει η OpenGL είναι πάρα πολύ χρήσιμη. Ενδεικτικά, χρησιμοποιώντας textures ως color buffers μπορούμε να δημιουργήσουμε καθρέπτες ή αντανακλάσεις στην επιφάνεια νερού ή να εφαρμόσουμε post-processing effects, ενώ χρησιμοποιώντας textures ως depth buffers μπορούμε να υλοποιήσουμε τη διασημότερη τεχνική δημιουργίας σκιών, τη χαρτογράφηση σκιών (shadow mapping).
